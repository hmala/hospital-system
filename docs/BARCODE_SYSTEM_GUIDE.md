# نظام الباركود المتقدم - دليل الاستخدام

## ✅ التحديثات المنفذة

### 1. إضافة حقول جديدة لجدول `stock_batches`
- `supplier_barcode`: باركود المورد/الشركة المصنعة
- `manufacturer_lot_number`: رقم دفعة المصنع

### 2. تحديث نموذج البيانات
- `StockBatch` Model محدّث بالحقول الجديدة
- `PurchaseController` يدعم حفظ البيانات الجديدة
- Validation للحقول الجديدة (اختيارية)

### 3. واجهات المستخدم
- ✅ تحديث `/purchases/create` لإدخال باركود المورد ورقم الدفعة
- ✅ واجهات طباعة الباركود:
  - `/barcodes` - قائمة الدفعات
  - `/barcodes/batch/{id}` - طباعة دفعة واحدة
  - `/barcodes/purchase/{id}` - طباعة باركودات فاتورة كاملة
  - `/barcodes/print-multiple` - طباعة باركودات متعددة

### 4. نظام طباعة QR Code و Barcode
- استخدام مكتبة `qrcode.js` لتوليد QR codes
- استخدام مكتبة `jsbarcode` لتوليد باركود خطي (CODE128)
- دعم طباعة مباشرة من المتصفح
- ملصقات جاهزة للطباعة على ورق A4

## 📝 كيفية الاستخدام

### السيناريو 1: إنشاء مشتريات جديدة مع الباركود

1. اذهب إلى `/purchases/create`
2. اختر المورد ورقم الفاتورة
3. أضف المواد:
   - اختر المادة
   - أدخل الكمية والسعر
   - (اختياري) أدخل **باركود المورد** إذا كان موجود على العبوة
   - (اختياري) أدخل **رقم دفعة المصنع** من العبوة
   - إذا كانت المادة قابلة للتلف، أدخل تاريخ الانتهاء
4. احفظ الفاتورة

### السيناريو 2: طباعة باركودات الفاتورة

1. بعد حفظ الفاتورة، ستظهر صفحة التفاصيل
2. اضغط على زر **"طباعة الباركودات"**
3. ستفتح صفحة جديدة بها:
   - QR Code لكل مادة (يحتوي على كل البيانات)
   - Barcode خطي (CODE128) للمسح السريع
   - معلومات المادة (الاسم، الكمية، تاريخ الانتهاء، رقم الدفعة)
4. اطبع الصفحة (Ctrl+P أو زر الطباعة)
5. قص الملصقات والصقها على العبوات

### السيناريو 3: طباعة باركودات متعددة

1. اذهب إلى `/barcodes`
2. اختر الدفعات المطلوبة (checkbox)
3. اضغط "طباعة المحدد"
4. ستفتح صفحة بكل الباركودات المحددة
5. اطبع

## 🔍 محتوى QR Code

يحتوي QR Code على البيانات التالية بصيغة JSON:

```json
{
  "batch_id": 15,
  "product_id": 5,
  "internal_barcode": "B5-260406-ABCD",
  "product_name": "باراسيتامول 500mg",
  "expiry_date": "2027-12-31",
  "supplier_barcode": "123456789",
  "lot_number": "LOT-2024-001"
}
```

### فائدة QR Code
- مسح ضوئي بالموبايل يعطي كل البيانات
- يمكن استخدامه في تطبيقات الموبايل لتتبع المخزون
- يحتوي على معلومات أكثر من الباركود الخطي

## 🎯 المزايا الرئيسية

### 1. نظام ثلاثي المستويات
- **المستوى 1**: باركود المنتج (`PRD-0001`)
- **المستوى 2**: باركود الدفعة الداخلي (`B5-260406-ABCD`)
- **المستوى 3**: باركود المورد + رقم دفعة المصنع

### 2. تتبع كامل
- تتبع أصل الدفعة عبر `original_barcode`
- تتبع دفعة المصنع عبر `manufacturer_lot_number`
- تتبع حركة المخزون بين المواقع

### 3. FIFO (First In, First Out)
- استخدام `received_at` لتحديد الأقدم
- `original_received_at` يحفظ تاريخ الاستلام الأصلي

### 4. التحقق من الأصالة
- باركود المورد للتحقق من الشركة المصنعة
- رقم الدفعة للاستعلام عن قواعد بيانات المصنع

## 🧪 الاختبار

### خطوات الاختبار الأساسية:

1. **إنشاء مشتريات:**
   ```
   - اذهب لـ /purchases/create
   - أضف مادة مع باركود مورد: "TEST123456"
   - أضف رقم دفعة: "LOT-2024-TEST"
   - احفظ
   ```

2. **طباعة الباركود:**
   ```
   - افتح تفاصيل الفاتورة
   - اضغط "طباعة الباركودات"
   - تحقق من ظهور QR Code و Barcode
   ```

3. **مسح QR Code:**
   ```
   - استخدم تطبيق QR Scanner على الموبايل
   - امسح الكود المطبوع
   - تحقق من البيانات المعروضة
   ```

### توقعات النجاح:
- ✅ الباركود الداخلي يُنشأ تلقائياً بصيغة `B{productId}-{date}-{random}`
- ✅ باركود المورد ورقم الدفعة يُحفظان في قاعدة البيانات
- ✅ QR Code قابل للمسح ويحتوي على JSON صحيح
- ✅ Barcode خطي قابل للمسح بأجهزة Scanner
