    نظام السعر المخصص للعمليات المحولة من خارج المستشفى
═══════════════════════════════════════════════════════════════

📋 الوصف:
────────────────────────────────────────────────────────────

تم إضافة ميزة جديدة تسمح بإدخال سعر مخصص للعمليات الجراحية
المحولة من أطباء خارجيين (خارج المستشفى).


⚙️ كيفية العمل:
────────────────────────────────────────────────────────────

1. عند إنشاء عملية جراحية جديدة
2. إذا اخترت "طبيب خارجي" في حقل الطبيب المرسل
3. سيظهر حقل جديد: "سعر العملية (تحويل خارجي)"
4. يجب إدخال السعر يدوياً من قبل موظف الاستعلامات


📊 الحالات المختلفة:
────────────────────────────────────────────────────────────

🔹 تحويل داخلي (من المستشفى):
   ✓ السعر يؤخذ تلقائياً من جدول العمليات
   ✓ يظهر تنبيه بالسعر حسب الجدول
   ✓ لا حاجة لإدخال سعر يدوياً

🔹 تحويل خارجي (طبيب خارجي):
   ✓ يظهر حقل "سعر العملية (تحويل خارجي)"
   ✓ يجب إدخال السعر يدوياً (إجباري)
   ✓ السعر متغير حسب الحالة


💡 ملاحظات مهمة:
────────────────────────────────────────────────────────────

• حقل السعر المخصص إجباري للتحويلات الخارجية فقط
• يمكن إدخال أي سعر (لا حد أدنى أو أقصى محدد)
• السعر يُحفظ مع بيانات العملية في قاعدة البيانات
• عند التحديث، يمكن تغيير السعر إذا لزم الأمر


🎯 الفائدة:
────────────────────────────────────────────────────────────

✓ مرونة في تسعير العمليات الخارجية
✓ لا يتقيد بجدول الأسعار الثابت
✓ موظف الاستعلامات يتحكم بالسعر حسب الحالة
✓ شفافية في التسعير


📁 الملفات المعدلة:
────────────────────────────────────────────────────────────

1. resources/views/surgeries/create.blade.php
   - إضافة حقل السعر المخصص
   - إضافة JavaScript للتحكم بالحقل
   - إظهار/إخفاء حسب نوع التحويل

2. app/Http/Controllers/SurgeryController.php
   - إضافة validation للسعر المخصص
   - تحديث منطق حساب السعر
   - دعم store() و update()
