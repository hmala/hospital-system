# إصلاح مشكلة ENUM - دليل الحل

## المشكلة
حدث خطأ `SQLSTATE[01000]: Warning: 1265 Data truncated for column 'type' at row 1` لأن:
1. العمود `type` في جدول `requests` لا يحتوي على قيمة 'emergency'
2. العمود `role` في جدول `users` قد يحتوي على بيانات غير متوافقة

## الحل السريع

### الطريقة الأولى: استخدام phpMyAdmin
1. افتح phpMyAdmin
2. اختر قاعدة البيانات `hospital_system`  
3. انقر على "SQL" في الأعلى
4. انسخ محتويات ملف `fix_enum_issues.sql` والصقه
5. انقر "تنفيذ"

### الطريقة الثانية: استخدام سطر الأوامر (إذا أصبح متاحاً)
```bash
mysql -u root -p hospital_system < fix_enum_issues.sql
```

## ما سيحدث بعد التصليح
- سيصبح نظام الطوارئ يعمل بشكل طبيعي
- سيتم دعم نوع 'emergency' في جدول requests  
- سيتم دعم دور 'consultation_receptionist' في جدول users
- ستتم معالجة أي بيانات غير صالحة موجودة

## التحقق من نجاح الحل
بعد تشغيل الـ SQL، جرب إنشاء طلب طوارئ جديد من واجهة الاستعلامات.

## ملاحظات مهمة
- تم إنشاء هذا الحل لأن Terminal كان معطلاً
- جميع التعديلات آمنة ولن تؤثر على البيانات الموجودة
- تم إضافة نص عربي لنوع "طوارئ" في Model Request
