## 1. الجزء الأول: دليل التثبيت والتهيئة للبيئة البرمجية

### 💻 متطلبات النظام الأساسية:
* **PHP**: إصدار 8.2 أو أحدث.
* **Laravel**: إصدار 12.x.
* **قاعدة البيانات**: MySQL أو MariaDB.
* **إدارة الحزم**: Composer للـ PHP و Node.js (مع npm) للـ Frontend.

### ⚙️ خطوات التثبيت المحلي:
1. **نسخ المشروع وتهيئة البيئة:**
   ```bash
   composer install
   npm install
   ```
2. **إعداد ملف التكوين البيئي:**
   قم بإنشاء ملف `.env` وقم بتهيئة الاتصال بقاعدة البيانات:
   ```env
   DB_CONNECTION=mysql
   DB_HOST=127.0.0.1
   DB_PORT=3306
   DB_DATABASE=hospital_system
   DB_USERNAME=root
   DB_PASSWORD=
   ```
3. **تنفيذ الهجرات وقواعد البيانات الأولية (Migrations & Seeders):**
   ```bash
   php artisan migrate --force
   php artisan db:seed --class=RolesAndPermissionsSeeder
   php artisan db:seed --class=IncubatorSeeder
   ```
4. **تشغيل خادم التطوير والـ Vite:**
   ```bash
   php artisan serve
   npm run dev
   ```

## 2. الجزء الثاني: إدارة الصلاحيات والأدوار وحماية الوصول (Spatie)

يعتمد النظام بالكامل على حزمة `spatie/laravel-permission` لتنظيم الأدوار والصلاحيات.

### 👥 الأدوار الثمانية وصلاحياتها الافتراضية:

1. **المدير (admin):** صلاحية كاملة ومطلقة على جميع مسارات وواجهات النظام.
2. **الطبيب (doctor):**
   * عرض المرضى والزيارات المخصصة له (`view patients`, `manage own visits`).
   * طلب الفحوصات والعمليات (`create radiology`, `create lab tests`, `create surgeries`).
3. **المريض (patient):**
   * عرض زياراته الخاصة فقط (`view own visits`).
   * حجز وإلغاء المواعيد (`create appointments`, `cancel appointments`).
4. **موظف الاستقبال (receptionist):**
   * حجز المواعيد وإدارة الاستعلامات والزيارات والتحويلات للأقسام.
5. **موظف المختبر (lab_staff):**
   * عرض ومعالجة طلبات التحاليل وإدخال نتائج الفحوصات (`process lab requests`).
6. **موظف الأشعة (radiology_staff):**
   * عرض ومعالجة طلبات الأشعة وإدخال التقارير والصور (`process radiology requests`).
7. **موظف الصيدلية (pharmacy_staff):**
   * معالجة وصرف الأدوية للمرضى (`process pharmacy requests`).
8. **موظف العمليات (surgery_staff):**
   * متابعة قائمة انتظار العمليات والتحكم بمحطات العمليات.

### 🔐 صلاحيات حجز الأشعة الفرعية المخصصة (Radiology Inquiry Permissions):
تم تقسيم حجز الأشعة من الاستعلامات إلى 4 صلاحيات دقيقة لمنع التداخل وحماية الخصوصية:
* `inquiry.create.radiology.general` (الأشعة العامة)
* `inquiry.create.radiology.ultrasound` (السونار)
* `inquiry.create.radiology.mri` (الرنين المغناطيسي)
* `inquiry.create.radiology.echo` (الإيكو)
* `inquiry.create.radiology` (صلاحية عامة لكافة الأنواع - للرجوع والتوافق)

#### 💻 أوامر Tinker لإدارة صلاحيات الأشعة:
```php
// إعطاء دور الاستقبال صلاحية حجز السونار فقط
$role = \Spatie\Permission\Models\Role::where('name', 'receptionist')->first();
$role->givePermissionTo('inquiry.create.radiology.ultrasound');

// إعطاء صلاحية الرنين والإيكو لمستخدم محدد بالبريد
$user = \App\Models\User::where('email', 'receptionist@hospital.com')->first();
$user->givePermissionTo(['inquiry.create.radiology.mri', 'inquiry.create.radiology.echo']);

// إعادة تعيين وحذف الكاش بعد التحديث
php artisan permission:cache-reset
```

## 3. الجزء الثالث: نظام استعلامات المرضى وحجز الفحوصات المتعددة

### 📋 سير العمل لإنشاء الطلبات من الاستعلامات:
1. يذهب الموظف إلى شاشة الاستعلامات الرئيسية (`/inquiry`).
2. يبحث عن المريض بالاسم أو الهاتف باستخدام شريط البحث السريع المعتمد على **AJAX**.
3. إذا لم يكن للمريض ملف سابق، يتم الضغط على "إضافة مريض جديد".
4. تظهر **4 بطاقات ملونة** لاختيار الخدمة المطلوبة:
   * 🔵 **تحاليل طبية (Lab)**: تظهر قائمة بـ 398 تحليلاً مقسمة حسب الفئات.
   * 🔷 **أشعة (Radiology)**: تظهر قائمة بـ 185 فحصاً إشعاعياً مصنفاً.
   * 🟢 **صيدلية (Pharmacy)**: لطلب الأدوية.
   * 🟡 **كشف طبي (Checkup)**: لاختيار الطبيب والعيادة وتحديد موعد الكشف.

### 🔄 ميزة الاختيار المتعدد والتحويل التلقائي (Multi-Selection & Auto-Referral):
* يمكن للموظف اختيار أكثر من خدمة معاً (مثل تحاليل + أشعة + كشف) في نموذج واحد.
* عند تفعيل خيار **"التحويل التلقائي"**، يتم توجيه الموظف بعد الحفظ مباشرة إلى شاشة التحويل لتحديد التواريخ والأقسام دون الحاجة للعودة للقائمة الرئيسية.
* يتم حفظ المعرفات المختارة (تحاليل أو أشعة) في عمود الـ `details` بجدول `requests` بصيغة JSON:
  ```json
  {
    "lab_test_ids": [1, 5, 12],
    "radiology_type_ids": [3, 7]
  }
  ```

## 4. الجزء الرابع: نظام الصندوق والمدفوعات والمحاسبة المالية

يمتنع تقديم أي خدمة للمريض (كشف، تحاليل، أشعة) إلا بعد تسديد رسومها المقررة عند الكاشير.

### 📊 هيكلية قاعدة بيانات المدفوعات:

#### 1. جدول `payments` (جدول تسجيل الفواتير):
* `receipt_number`: رقم الإيصال الفريد بتنسيق `REC-YYYYMMDD-####`.
* `amount`: المبلغ المدفوع بالدينار العراقي.
* `payment_method`: طريقة الدفع (`cash`, `card`, `insurance`).
* `payment_type`: نوع الخدمة (`appointment`, `lab`, `radiology`, `surgery`, `pharmacy`, `other`).

#### 2. تعديلات جداول `appointments` و `requests`:
* إضافة عمود `payment_status` (`pending`, `paid`, `refunded`) لتتبع حالة الفاتورة.
* إضافة عمود `payment_id` لربط السجل بعملية الدفع المسجلة.

### 🩺 سير عمل الدفع لطلبات الطبيب (Doctor Request Payment Flow):
1. الطبيب يطلب تحاليل أو أشعة للمريض من شاشة معاينة الزيارة.
2. يتم إنشاء الطلب بحالة دفع `pending` ولا يظهر في شاشات المختبر أو الأشعة إطلاقاً لمنع التلاعب.
3. يظهر للطبيب والمريض تنبيه بالتوجه إلى شباك الكاشير.
4. يفتح الكاشير شاشته المخصصة (`/cashier`) ويجد قسمين: "المواعيد المعلقة" و "الطلبات الطبية المعلقة".
5. يقوم الكاشير بالضغط على زر **تسديد** للملف المطلوب، ويختار طريقة الدفع ثم يؤكد السداد.
6. تتحدث حالة الطلب برمجياً إلى `paid` فوراً، وينشأ سجل الدفع، وتنتقل الحالة للقسم المختص تلقائياً.

### 💰 السعر المخصص للعمليات الجراحية الخارجية:
* عند إدخال عملية جراحية لمريض محول من **طبيب خارجي**، يظهر حقل إلزامي إضافي: **"سعر العملية (تحويل خارجي)"**.
* يتيح هذا الحقل إدخال سعر مخصص متفق عليه يدوياً ولا يتقيد بأسعار العمليات الثابتة الخاصة بمرضى الأطباء الداخليين.

### 🔔 نظام الإشعارات الفورية للمدفوعات (Real-time Notifications):
* يتم دمج مكتبة Laravel Echo و Pusher لإرسال تنبيه فوري فور سداد الفاتورة:
  * يظهر تنبيه فوري في لوحة الكاشير: `💰 موعد جديد بانتظار الدفع`.
  * يظهر تنبيه فوري لموظف الاستعلامات: `✅ تم تسديد موعد المريض [X]`.

## 5. الجزء الخامس: نظام الأرشفة التلقائية وجسر السكانر المحلي

يسمح النظام بسحب وثائق تحويل المرضى من أجهزة السكانر المتصلة بالحاسوب بضغطة زر واحدة داخل المتصفح.

### 🏗️ بنية وعمل الجسر البرمجي المحلي (Flask Python Bridge):
```
المنظومة:
متصفح العميل ──(HTTP POST /scan)──> خادم Flask محلي (localhost:37426) ──> استدعاء CLI لبرنامج NAPS2 ──> السكانر الفعلي
```

### 📥 روابط تحميل البرامج المساعدة:
1. **Python 3.8+**: [رابط التحميل](https://www.python.org/downloads/) (مهم: تفعيل خيار Add Python to PATH أثناء التثبيت).
2. **NAPS2 (Not Another PDF Scanner)**: [رابط التحميل المباشر](https://github.com/cyanfish/naps2/releases/latest) (يجب تثبيته في المسار الافتراضي `C:\Program Files\NAPS2\`).

### 🚀 تشغيل النظام (موظف الأرشفة):
* انقر نقراً مزدوجاً على الملف `📄 تشغيل_برنامج_الارشفة.bat` الموجود في المجلد الرئيسي للمشروع.
* ستظهر نافذة سوداء (Command Prompt)، اتركها مفتوحة طوال فترة العمل. ستعرض الرسالة التالية:
  ```
  ✓ الخادم يعمل على: http://localhost:37426
  ```
* ضع الورقة في السكانر واضغط على زر **"مسح الورقة من السكانر الآن"** في صفحة حجز العملية بالمتصفح ليتم سحب الصورة تلقائياً.

### 🔧 مسارات API السكانر (لوحة البايثون):
* `POST /scan`: يقوم بتهيئة السكانر، وسحب الصورة بصيغة JPEG بجودة عالية وإرسالها كـ Binary Blob للمتصفح.
* `GET /status`: يتحقق من جاهزية خادم الأرشفة المحلي ويعيد رد JSON:
  ```json
  {"status": "ready", "message": "جاهز للمسح"}
  ```

## 6. الجزء السادس: محطات العمليات الجراحية وحاضنات الخدج ICU

### 🩺 محطات العمليات الجراحية الست المتسلسلة (Surgery Stations):
تنتقل حالة العملية جراحياً بين المحطات بالترتيب الإلزامي التالي، ولا يمكن الانتقال لمحطة إلا بعد إتمام المحطة السابقة:

```
1. حجز العملية (الاستعلامات) 
       ↓
2. المقيم قبل العملية (Pre-Op) ──> فحص وتأكيد جاهزية المريض
       ↓
3. صالة العمليات (Operation Theater) ──> تعيين طبيب التخدير وممرض الصالة والأوقات
       ↓
4. الطبيب الجراح (Surgeon Station) ──> توثيق تفاصيل العملية والتشخيص النهائي
       ↓
5. محطة التخدير (Anesthesia Station) ──> توثيق تفاصيل التخدير والمساعدين
       ↓
6. المقيم بعد العملية (Post-Op) ──> المتابعة وتوصيات العلاج وتحديد المراجعة
       ↓
7. محطة التمريض (Nursing Station) ──> العلامات الحيوية النهائية وتصريح الخروج (تتحول الحالة إلى Completed)
```

### 👶 نظام حاضنات الأطفال الخدج (NICU ICU System):
* يحتوي قسم العناية المركزة بالخدج (ط6) على **16 حاضنة** موزعة كالتالي:
  * **الحاضنات العادية (1-6):** الأجرة اليومية 50,000 د.ع.
  * **حاضنات + أكسجين (7-11):** الأجرة اليومية 75,000 د.ع.
  * **حاضنات + علاج ضوئي (12-16):** الأجرة اليومية 80,000 د.ع.
* الحالات المتاحة للحاضنة: `available` (متاحة)، `occupied` (مشغولة)، `maintenance` (صيانة).
* حساب التكلفة نهائياً وتلقائياً عند خروج الطفل بناءً على عدد الأيام الفعلي لربط الحجز بالخروج.

## 7. الجزء السابع: نظام الباركود المتقدم وتتبع المخزون

يدعم النظام إدارة المخازن عبر نظام باركود متطور ثلاثي المستويات لتتبع حركة المواد الطبية والأدوية.

### 🏷️ الهيكلية الثلاثية المستويات للباركود:
1. **المستوى 1 (باركود المادة):** باركود ثابت لكل منتج (مثال: `PRD-0001`).
2. **المستوى 2 (الباركود الداخلي للدفعة):** باركود فريد يولد تلقائياً لكل دفعة مشتريات مستلمة بتنسيق `B{productId}-{date}-{random}` لتتبع دفعات الصلاحية (FIFO).
3. **المستوى 3 (باركود المورد وتفاصيل المصنع):**
   * `supplier_barcode`: الباركود المطبوع على عبوة المنتج الأصلية من المورد.
   * `manufacturer_lot_number`: رقم الدفعة (Lot Number) الخاص بالمصنع للتحقق من سلامة الأدوية.

### 🔍 تفاصيل ومكونات QR Code المطبوع:
يتم توليد رمز استجابة سريعة (QR Code) يحتوي على مصفوفة JSON متكاملة لتسهيل مسحها بالهواتف الذكية أو أجهزة المسح المحمولة:
```json
{
  "batch_id": 15,
  "product_id": 5,
  "internal_barcode": "B5-260406-ABCD",
  "product_name": "Paracetamol 500mg",
  "expiry_date": "2027-12-31",
  "supplier_barcode": "123456789",
  "lot_number": "LOT-2024-001"
}
```

## 8. الجزء الثامن: ملاحق المطور البرمجية وحل المشاكل

### 🛠️ حل مشكلة ENUM لطلبات الطوارئ (Data Truncated):
إذا ظهر خطأ `SQLSTATE[01000]: Warning: 1265 Data truncated for column 'type'` بسبب عدم احتواء عمود `type` في جدول `requests` على قيمة `emergency` أو `consultation_receptionist` في جدول المستخدمين، قم بتشغيل الاستعلام التالي في MySQL مباشرة لحل المشكلة:

```sql
-- تحديث عمود النوع في جدول الطلبات لدعم الطوارئ
ALTER TABLE requests MODIFY COLUMN type ENUM('lab', 'radiology', 'pharmacy', 'checkup', 'emergency') NOT NULL;

-- تحديث أدوار المستخدمين لدعم موظف استقبال الاستشارات
ALTER TABLE users MODIFY COLUMN role ENUM('admin', 'doctor', 'patient', 'receptionist', 'lab_staff', 'radiology_staff', 'pharmacy_staff', 'surgery_staff', 'cashier', 'consultation_receptionist') NOT NULL;
```

### 💻 أوامر Laravel Tinker مفيدة ومستعملة بكثرة:
```bash
# تشغيل بيئة Tinker التفاعلية
php artisan tinker

# 1. إظهار كافة المستخدمين المعين لهم دور محدد
\App\Models\User::role('cashier')->get();

# 2. تغيير كلمة مرور مستخدم بشكل يدوي وسريع
$user = \App\Models\User::where('email', 'admin@hospital.com')->first();
$user->password = Hash::make('new_secure_password');
$user->save();

# 3. التحقق من صلاحيات مستخدم محدد
$user->getAllPermissions()->pluck('name');
```
